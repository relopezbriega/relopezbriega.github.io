Machine Learning con Python - Sobreajuste

Sun 29 May 2016

Tweet

Introducción¶

Uno de los conceptos ms importantes en Machine Learning es el overfitting o sobreajuste del modelo. Comprender como un modelo se ajusta a los datos es muy importante para entender las causas de baja precisin en las predicciones. Un modelo va a estar sobreajustado cuando vemos que se desempea bien con los datos de entrenamiento, pero su precisin es notablemente ms baja con los datos de evaluacin; esto se debe a que el modelo ha memorizado los datos que ha visto y no pudo generalizar las reglas para predecir los datos que no ha visto. De aqu tambin la importancia de siempre contar con dos conjuntos de datos distintos, uno para entrenar el modelo y otro para evaluar su precisin; ya que si utilizamos el mismo dataset para las dos tareas, no tendramos forma de determinar como el modelo se comporta con datos que nunca ha visto.

¿Cómo reconocer el sobreajuste?¶

En lneas generales el sobreajuste va a estar relacionado con la complejidad del modelo, mientras ms complejidad le agreguemos, mayor va a ser la tendencia a sobreajustarse a los datos, ya que va a contar con mayor flexibilidad para realizar las predicciones y puede ser que los patrones que encuentre estn relacionados con el ruido (pequeos errores aleatorios) en los datos y no con la verdadera seal o relacin subyacente.

No existe una regla general para establecer cual es el nivel ideal de complejidad que le podemos otorgar a nuestro modelo sin caer en el sobreajuste; pero podemos valernos de algunas herramientas analticas para intentar entender como el modelo se ajusta a los datos y reconocer el sobreajuste. Veamos un ejemplo.

Árboles de Decisión y sobreajuste¶

Los rboles de Decisin pueden ser muchas veces una herramienta muy precisa, pero tambin con mucha tendencia al sobreajuste. Para construir estos modelos aplicamos un procedimiento recursivo para encontrar los atributos que nos proporcionan ms informacin sobre distintos subconjuntos de datos, cada vez ms pequeos. Si aplicamos este procedimiento en forma reiterada, eventualmente podemos llegar a un rbol en el que cada hoja tenga una sola instancia de nuestra variable objetivo a clasificar. En este caso extremo, el rbol de Decisin va a tener una pobre generalizacin y estar bastante sobreajustado; ya que cada instancia de los datos de entrenamiento va a encontrar el camino que lo lleve eventualmente a la hoja que lo contiene, alcanzando as una precisin del 100% con los datos de entrenamiento. Veamos un ejemplo sencillo con la ayuda de Python.

Métodos para reducir el Sobreajuste¶

Algunas de las tcnicas que podemos utilizar para reducir el Sobreajuste, son:

  • Utilizar validacin cruzada.
  • Recolectar ms datos.
  • Introducir una penalizacin a la complejidad con alguna tcnica de regularizacin.
  • Optimizar los parmetros del modelo con grid search.
  • Reducir la dimensin de los datos.
  • Aplicar tcnicas de seleccin de atributos.
  • Utilizar modelos ensamblados.

Veamos algunos ejemplos.

Validación cruzada¶

La validacin cruzada se inicia mediante el fraccionamiento de un conjunto de datos en un nmero $k$ de particiones (generalmente entre 5 y 10) llamadas pliegues. La validacin cruzada luego itera entre los datos de evaluacin y entrenamiento $k$ veces, de un modo particular. En cada iteracin de la validacin cruzada, un pliegue diferente se elige como los datos de evaluacin. En esta iteracin, los otros pliegues $k-1$ se combinan para formar los datos de entrenamiento. Por lo tanto, en cada iteracin tenemos $(k-1) / k$ de los datos utilizados para el entrenamiento y $1 / k$ utilizado para la evaluacin. Cada iteracin produce un modelo, y por lo tanto una estimacin del rendimiento de la generalizacin, por ejemplo, una estimacin de la precisin. Una vez finalizada la validacin cruzada, todos los ejemplos se han utilizado slo una vez para evaluar pero $k -1$ veces para entrenar. En este punto tenemos estimaciones de rendimiento de todos los pliegues y podemos calcular la media y la desviacin estndar de la precisin del modelo. Veamos un ejemplo

In [13]:
# Ejemplo cross-validation
from sklearn import cross_validation

# creando pliegues
kpliegues = cross_validation.StratifiedKFold(y=y_train, n_folds=10,
                                        random_state=2016)
# iterando entre los plieges
precision = []
for k, (train, test) in enumerate(kpliegues):
    arbol2.fit(x_train[train], y_train[train]) 
    score = arbol2.score(x_train[test], y_train[test])
    precision.append(score)
    print('Pliegue: {0:}, Dist Clase: {1:}, Prec: {2:.3f}'.format(k+1,
                        np.bincount(y_train[train]), score))

# imprimir promedio y desvio estandar
print('Precision promedio: {0: .3f} +/- {1: .3f}'.format(np.mean(precision),
                                          np.std(precision)))

Más datos y curvas de aprendizaje¶

Otra herramienta analtica que nos ayuda a entender como reducimos el Sobreajuste con la ayuda de ms datos, son las curvas de aprendizaje, las cuales grafican la precisin en funcin del tamao de los datos de entrenamiento. Veamos como podemos graficarlas con la ayuda de Python.

Optimización de parámetros con Grid Search¶

La mayora de los modelos de Machine Learning cuentan con varios parmetros para ajustar su comportamiento, por lo tanto otra alternativa que tenemos para reducir el Sobreajuste es optimizar estos parmetros por medio de un proceso conocido como grid search e intentar encontrar la combinacin ideal que nos proporcione mayor precisin. El enfoque que utiliza grid search es bastante simple, se trata de una bsqueda exhaustiva por el paradigma de fuerza bruta en el que se especifica una lista de valores para diferentes parmetros, y la computadora evala el rendimiento del modelo para cada combinacin de stos parmetros para obtener el conjunto ptimo que nos brinda el mayor rendimiento.

Veamos un ejemplo utilizando un modelo de SVM o Mquinas de vectores de soporte, la idea va a ser optimizar los parmetros gamma y C de este modelo. El parmetro gamma define cuan lejos llega la influencia de un solo ejemplo de entrenamiento, con valores bajos que significan "lejos" y los valores altos significan "cerca". El parmetro C es el que establece la penalizacin por error en la clasificacin un valor bajo de este parmetro hace que la superficie de decisin sea ms lisa, mientras que un valor alto tiene como objetivo que todos los ejemplos se clasifiquen correctamente, dndole ms libertad al modelo para elegir ms ejemplos como vectores de soporte. Tengan en cuenta que como todo proceso por fuerza bruta, puede tomar bastante tiempo segn la cantidad de parmetros que utilicemos para la optimizacin.

In [17]:
# Ejemplo de grid search con SVM.
from sklearn.grid_search import GridSearchCV

# creacin del modelo
svm = SVC(random_state=1982)

# rango de parametros
rango_C = np.logspace(-2, 10, 10)
rango_gamma = np.logspace(-9, 3, 10)
param_grid = dict(gamma=rango_gamma, C=rango_C)

# crear grid search
gs = GridSearchCV(estimator=svm, param_grid=param_grid, scoring='accuracy',
                  cv=5,n_jobs=-1)

# comenzar el ajuste
gs = gs.fit(x_train, y_train)
